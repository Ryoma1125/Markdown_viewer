# 設計パターン・アーキテクチャ

## 概要

ソフトウェア設計の原則とパターンを学び、保守性・拡張性の高いコードを書けるようになるカリキュラムです。

## 前提知識

- オブジェクト指向プログラミングの基本（クラス、継承、インターフェース）
- Python または Java の基本的な文法

## 学習期間の目安

- 1日1〜2時間: 約4週間
- 集中学習: 約2週間

## カリキュラム構成

```
Design-Patterns/
├── README.md（このファイル）
├── phase1/
│   ├── 01_SOLID原則.md          # 設計の基本原則
│   └── 02_クリーンコード.md       # 読みやすいコードの書き方
├── phase2/
│   ├── 01_生成パターン.md         # Factory, Singleton, Builder
│   └── 02_構造パターン.md         # Adapter, Decorator, Facade
├── phase3/
│   ├── 01_振る舞いパターン.md     # Strategy, Observer, Command
│   └── 02_よく使うパターン実践.md  # Repository, DTO, DI
├── phase4/
│   ├── 01_レイヤードアーキテクチャ.md
│   └── 02_クリーンアーキテクチャ.md
├── phase5/
│   └── 01_総仕上げ.md            # リファクタリング実践
```

## 到達目標

- SOLID 原則を説明・適用できる
- 主要なデザインパターンを使い分けられる
- アーキテクチャの選択肢を説明できる
- 既存コードをリファクタリングできる
