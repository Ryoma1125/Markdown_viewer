# 分散システム設計

## 概要

大規模システムを支える分散システムの設計原則を学ぶカリキュラムです。スケーラビリティ、可用性、一貫性のトレードオフを理解し、アーキテクトとしての意思決定ができるようになります。

## 前提知識

- ネットワーク基礎
- データベース基礎
- API 設計

## 学習期間の目安

- 1日1〜2時間: 約5日
- 集中学習: 約3日

## カリキュラム構成

```
Distributed-Systems/
├── README.md（このファイル）
├── phase1/
│   ├── 01_分散システム入門.md        # なぜ分散か
│   └── 02_CAP定理.md                # トレードオフ
├── phase2/
│   ├── 01_一貫性モデル.md            # 強一貫性、結果整合性
│   └── 02_レプリケーション.md         # データ複製
├── phase3/
│   ├── 01_合意アルゴリズム.md         # Raft, Paxos
│   └── 02_分散トランザクション.md     # 2PC, Saga
├── phase4/
│   └── 01_総仕上げ.md                # 設計演習
```

## 到達目標

- 分散システムの基本概念を説明できる
- CAP 定理を理解し設計に活かせる
- 一貫性と可用性のトレードオフを判断できる
- 分散トランザクションを設計できる
