# データベース基礎

## 概要

リレーショナルデータベース（RDB）の基礎から実践までを体系的に学ぶカリキュラムです。

## 前提知識

- Linux 基本操作（ファイル操作、ターミナル）
- Docker の基本（docker run, docker compose）

## 学習期間の目安

- 1日1〜2時間: 約3〜4週間
- 集中学習: 約2週間

## カリキュラム構成

```
Database/
├── README.md（このファイル）
├── phase0/
│   └── 00_環境構築.md        # Docker で MySQL 環境構築
├── phase1/
│   └── 01_データベースとは.md # RDB の概念、テーブル構造
├── phase2/
│   ├── 01_SELECT基礎.md      # データ取得の基本
│   └── 02_データ操作.md       # INSERT, UPDATE, DELETE
├── phase3/
│   ├── 01_テーブル設計.md     # CREATE TABLE, データ型
│   └── 02_正規化とリレーション.md # 正規化、外部キー
├── phase4/
│   ├── 01_JOIN操作.md        # 結合クエリ
│   └── 02_集計と分析.md       # GROUP BY, サブクエリ
├── phase5/
│   ├── 01_インデックス.md     # パフォーマンス最適化
│   └── 02_トランザクション.md  # ACID、ロック
├── phase6/
│   └── 01_総仕上げ.md        # 実践プロジェクト
```

## AWS との対応

| 学習内容 | AWS サービス |
|---------|-------------|
| MySQL | RDS (MySQL/Aurora) |
| データベース設計 | RDS スキーマ設計 |
| パフォーマンス | RDS Performance Insights |
| バックアップ | RDS スナップショット |

## 学習の進め方

1. 各 Phase を順番に進める
2. ハンズオンは必ず手を動かして実行
3. 理解度確認で知識を定着

## 到達目標

- SQL を使ってデータ操作ができる
- 適切なテーブル設計ができる
- パフォーマンスを考慮した実装ができる
- トランザクションを正しく扱える
