# DDD（ドメイン駆動設計）

## 概要

複雑なビジネスロジックを扱うためのドメイン駆動設計（DDD）を学ぶカリキュラムです。アーキテクト・テックリードを目指すあなたに必須の設計思想です。

## 前提知識

- オブジェクト指向プログラミング
- クラス設計の基本
- 設計パターンの基礎

## 学習期間の目安

- 1日1〜2時間: 約6日
- 集中学習: 約4日

## カリキュラム構成

```
DDD/
├── README.md（このファイル）
├── phase1/
│   ├── 01_DDD概要.md                # なぜDDDか
│   └── 02_ユビキタス言語.md          # 共通言語の重要性
├── phase2/
│   ├── 01_エンティティと値オブジェクト.md  # 基本ビルディングブロック
│   └── 02_集約.md                    # 集約とルート
├── phase3/
│   ├── 01_リポジトリ.md              # 永続化の抽象化
│   └── 02_ドメインサービス.md         # ドメインロジック
├── phase4/
│   ├── 01_境界づけられたコンテキスト.md  # コンテキスト分割
│   └── 02_コンテキストマップ.md        # コンテキスト間の関係
└── phase5/
    └── 01_総仕上げ.md                 # 実践プロジェクト
```

## 到達目標

- DDD の基本概念を説明できる
- エンティティと値オブジェクトを設計できる
- 集約とリポジトリを実装できる
- マイクロサービスの境界を定義できる
