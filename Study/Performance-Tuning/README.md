# パフォーマンスチューニング

## 概要

Web アプリケーションのパフォーマンス改善手法を学ぶカリキュラムです。ボトルネック特定から最適化まで実践的に学びます。

## 前提知識

- Python / Web アプリケーションの基本
- SQL / データベースの基本
- Docker の基本

## 学習期間の目安

- 1日1〜2時間: 約8日
- 集中学習: 約5日

## カリキュラム構成

```
Performance-Tuning/
├── README.md（このファイル）
├── phase1/
│   ├── 01_パフォーマンス基礎.md        # 指標、計測
│   └── 02_プロファイリング.md          # ボトルネック特定
├── phase2/
│   ├── 01_データベース最適化.md        # クエリ最適化
│   └── 02_インデックス設計.md          # インデックス戦略
├── phase3/
│   ├── 01_キャッシュ戦略.md            # 多層キャッシュ
│   └── 02_N+1問題.md                   # ORM の落とし穴
├── phase4/
│   ├── 01_非同期処理.md                # async/await
│   └── 02_並列処理.md                  # マルチプロセス
└── phase5/
    └── 01_総仕上げ.md                  # 実践プロジェクト
```

## 到達目標

- ボトルネックを特定できる
- データベースクエリを最適化できる
- キャッシュ戦略を設計できる
- 非同期処理を活用できる
