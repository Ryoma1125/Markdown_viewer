# API 設計（REST / GraphQL）

## 概要

Web API の設計原則を学び、RESTful API と GraphQL の両方を理解・実装できるようになるカリキュラムです。

## 前提知識

- HTTP の基本（メソッド、ステータスコード）
- Python の基本
- JSON の基本

## 学習期間の目安

- 1日1〜2時間: 約2週間
- 集中学習: 約1週間

## カリキュラム構成

```
API-Design/
├── README.md（このファイル）
├── phase1/
│   ├── 01_API設計基礎.md          # HTTP、REST の概念
│   └── 02_RESTful設計.md          # REST の原則
├── phase2/
│   ├── 01_FastAPI入門.md          # FastAPI で REST 実装
│   └── 02_バリデーション.md        # Pydantic、エラー処理
├── phase3/
│   ├── 01_認証と認可.md           # JWT、OAuth 連携
│   └── 02_ドキュメント.md          # OpenAPI、Swagger
├── phase4/
│   ├── 01_GraphQL入門.md          # GraphQL の概念
│   └── 02_GraphQL実装.md          # Strawberry で実装
└── phase5/
    └── 01_総仕上げ.md             # 実践 API 構築
```

## 到達目標

- RESTful API の設計原則を説明できる
- FastAPI で API を実装できる
- GraphQL の概念と利点を説明できる
- 適切な API ドキュメントを作成できる
