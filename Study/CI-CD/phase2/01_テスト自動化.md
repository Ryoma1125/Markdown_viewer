def add(a: int, b: int) -> int:

### 演習2: テスト実行ワークフロー

```yaml
# .github/workflows/test.yml
name: Test

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - name: Cache pip
        uses: actions/cache@v4
        with:
          path: ~/.cache/pip
          key: ${{ runner.os }}-pip-${{ hashFiles('requirements-dev.txt') }}
          restore-keys: |
            ${{ runner.os }}-pip-
      
      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements-dev.txt
      
      - name: Run tests
        run: pytest tests/ -v
```

### 演習3: Lint の追加

```yaml
# .github/workflows/lint.yml
name: Lint

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

  lint:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - name: Install dependencies
        run: pip install flake8 black mypy
      
      # Flake8 - スタイルチェック
      - name: Flake8
        run: flake8 src/ tests/ --max-line-length=100
      
      # Black - フォーマットチェック
      - name: Black
        run: black --check src/ tests/
      
      # MyPy - 型チェック
      - name: MyPy
        run: mypy src/ --ignore-missing-imports
```

### 演習4: テストカバレッジ

```yaml
# .github/workflows/test.yml（更新版）
name: Test with Coverage

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - name: Install dependencies
        run: |
          pip install -r requirements-dev.txt
      
      - name: Run tests with coverage
        run: |
          pytest tests/ \
            --cov=src \
            --cov-report=xml \
            --cov-report=term-missing \
            --cov-fail-under=80
      
      - name: Upload coverage to Codecov
        uses: codecov/codecov-action@v3
        with:
          files: ./coverage.xml
          fail_ci_if_error: true
```

### 演習5: 統合ワークフロー

```yaml
# .github/workflows/ci.yml
name: CI

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - run: pip install flake8 black
      - name: Flake8
        run: flake8 src/ tests/
      - name: Black
        run: black --check src/ tests/
  
  test:
    needs: lint
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ['3.10', '3.11', '3.12']
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
      
      - name: Install dependencies
        run: pip install -r requirements-dev.txt
      
      - name: Run tests
        run: pytest tests/ -v --cov=src --cov-report=xml
      
      - name: Upload coverage
        if: matrix.python-version == '3.11'
        uses: codecov/codecov-action@v3
```

### 演習6: PR へのコメント

```yaml
# .github/workflows/pr-comment.yml
name: PR Check

on:
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      
      - run: pip install pytest pytest-cov
      
      - name: Run tests
        id: test
        run: |
          pytest tests/ --cov=src --cov-report=xml
          echo "coverage=$(python -c "import xml.etree.ElementTree as ET; print(round(float(ET.parse('coverage.xml').getroot().attrib['line-rate']) * 100, 2))")" >> $GITHUB_OUTPUT
      
      - name: Comment on PR
        uses: actions/github-script@v7
        with:
          script: |
            github.rest.issues.createComment({
              issue_number: context.issue.number,
              owner: context.repo.owner,
              repo: context.repo.repo,
              body: `## Test Results ✅\n\nCoverage: **${{ steps.test.outputs.coverage }}%**`
            })
```

## ベストプラクティス

### テストの種類と実行タイミング

| テスト | 実行タイミング | 所要時間 |
|-------|--------------|---------|
| Lint | 全コミット | 数秒 |
| Unit Test | 全コミット | 数分 |
| Integration | PR, main push | 数分〜10分 |
| E2E | main push, 定期 | 10分以上 |

### 失敗時の対応

```yaml
- name: Run tests
  continue-on-error: true  # 失敗しても続行
  id: test
  run: pytest tests/

- name: Check test result
  if: steps.test.outcome == 'failure'
  run: |
    echo "Tests failed!"
    exit 1
```

## 理解度確認

### 問題

CI でテストを実行する際、カバレッジが 80% 未満の場合にビルドを失敗させる pytest のオプションはどれか。

**A.** `--cov-min=80`

**B.** `--cov-fail-under=80`

**C.** `--coverage-threshold=80`

**D.** `--min-coverage=80`

---

### 解答・解説

**正解: B**

```bash
pytest tests/ --cov=src --cov-fail-under=80
```

- `--cov-fail-under` でカバレッジの閾値を設定
- 閾値未満だとテストが失敗扱いになる
