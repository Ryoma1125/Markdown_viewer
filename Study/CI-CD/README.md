# CI/CD パイプライン

## 概要

継続的インテグレーション（CI）と継続的デリバリー（CD）の概念を学び、GitHub Actions を使った自動化パイプラインを構築できるようになるカリキュラムです。

## 前提知識

- Git/GitHub の基本操作
- Docker の基本
- YAML の基本構文

## 学習期間の目安

- 1日1〜2時間: 約2週間
- 集中学習: 約1週間

## カリキュラム構成

```
CI-CD/
├── README.md（このファイル）
├── phase1/
│   ├── 01_CI_CDとは.md             # 概念と用語
│   └── 02_GitHub_Actions入門.md     # ワークフローの基本
├── phase2/
│   ├── 01_テスト自動化.md           # ユニットテスト、Lint
│   └── 02_ビルド自動化.md           # Docker イメージビルド
├── phase3/
│   ├── 01_デプロイ自動化.md         # 環境へのデプロイ
│   └── 02_実践パイプライン.md       # 本格的なワークフロー
└── phase4/
    └── 01_総仕上げ.md              # 完全な CI/CD 構築
```

## 到達目標

- CI/CD の概念と利点を説明できる
- GitHub Actions でワークフローを作成できる
- テスト・ビルド・デプロイを自動化できる
- 安全なデプロイ戦略を設計できる
