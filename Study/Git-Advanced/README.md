# Git/GitHub 応用

## 概要

Git の基本操作は理解している前提で、チーム開発で必須となる応用テクニックを学ぶカリキュラムです。

## 前提知識

- git add, commit, push, pull の基本操作
- ブランチの基本概念

## 学習期間の目安

- 1日1〜2時間: 約2週間
- 集中学習: 約1週間

## カリキュラム構成

```
Git-Advanced/
├── README.md（このファイル）
├── phase1/
│   ├── 01_ブランチ戦略.md    # Git Flow, GitHub Flow
│   └── 02_マージ戦略.md      # merge vs rebase
├── phase2/
│   ├── 01_コンフリクト解決.md # 競合の解消
│   └── 02_履歴の書き換え.md   # rebase -i, amend
├── phase3/
│   ├── 01_PR運用.md          # レビュー、CI連携
│   └── 02_GitHub機能活用.md   # Issues, Projects, Actions
├── phase4/
│   └── 01_総仕上げ.md        # チーム開発シミュレーション
```

## 到達目標

- チーム開発でのブランチ戦略を選択・運用できる
- コンフリクトを自信を持って解決できる
- PR ベースの開発フローを実践できる
- GitHub の機能を活用してプロジェクト管理ができる
