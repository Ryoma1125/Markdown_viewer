# NoSQL 深掘り

## 概要

DynamoDB を中心に、NoSQL データベースの設計パターンと最適化を学ぶカリキュラムです。RDB との違いを理解し、適切なユースケースで NoSQL を活用できるようになります。

## 前提知識

- データベース基礎
- AWS 基礎（特に DynamoDB）
- Python プログラミング

## 学習期間の目安

- 1日1〜2時間: 約4日
- 集中学習: 約2日

## カリキュラム構成

```
NoSQL/
├── README.md（このファイル）
├── phase1/
│   ├── 01_NoSQL概論.md               # CAP定理、種類
│   └── 02_DynamoDB基礎.md            # テーブル設計
├── phase2/
│   ├── 01_シングルテーブル設計.md     # GSI/LSI
│   └── 02_アクセスパターン.md         # 実践パターン
├── phase3/
│   └── 01_総仕上げ.md                # 実践プロジェクト
```

## 到達目標

- NoSQL の適切なユースケースを判断できる
- DynamoDB のシングルテーブル設計ができる
- GSI/LSI を効果的に使える
- コスト効率の良い設計ができる
