# テスト戦略・品質保証

## 概要

ソフトウェアテストの基礎から実践的なテスト戦略まで学び、品質の高いコードを書けるようになるカリキュラムです。

## 前提知識

- Python の基本
- 関数・クラスの概念

## 学習期間の目安

- 1日1〜2時間: 約2週間
- 集中学習: 約1週間

## カリキュラム構成

```
Testing/
├── README.md（このファイル）
├── phase1/
│   ├── 01_テストの基礎.md          # テストの種類、テストピラミッド
│   └── 02_pytest入門.md            # pytest の基本
├── phase2/
│   ├── 01_単体テスト.md            # ユニットテストの書き方
│   └── 02_モック.md                # モックの使い方
├── phase3/
│   ├── 01_結合テスト.md            # 結合テスト、API テスト
│   └── 02_E2Eテスト.md             # End-to-End テスト
├── phase4/
│   ├── 01_TDD.md                   # テスト駆動開発
│   └── 02_カバレッジ.md            # コードカバレッジ
└── phase5/
    └── 01_総仕上げ.md              # 実践テスト戦略
```

## 到達目標

- テストピラミッドを説明できる
- pytest で効率的にテストを書ける
- モックを適切に使える
- TDD の流れを説明できる
